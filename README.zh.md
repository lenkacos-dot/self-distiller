# Self-Distiller — 自我蒸馏知识库 Skill

> 基于 **Karpathy LLM Wiki 模式**的特化实例，用于自我认知。
> 和 LLM Wiki 共用同一套架构（三层 + 增量工作流），只是领域不同：
> - LLM Wiki → 你**拥有的知识**（工具、配置、项目）
> - Self-Distiller → 你**是谁**（认知、决策、价值）
> 两者各自独立，互不依赖。
>
> **[English → README.md](./README.md)**

---

## 这是什么？

一个 Agent 无关的 Skill，让**任何 AI Agent**（Hermes / Claude Code / Crush / 本地 Ollama 等）持续编译并维护关于**你是谁**的结构化知识库。

- **输入**：日记、笔记、对话记录、任何自我相关材料
- **输出**：按维度组织的 Wiki 页面，含置信度、来源引用、矛盾标记
- **数据**：纯 Markdown + JSON，全存本地，不上传任何云端

---

## 和 LLM Wiki 的关系

| | LLM Wiki | Self-Distiller |
|---|---|---|
| 面向 | 你**拥有的知识** | 你**是谁** |
| 内容 | 工具配置、项目笔记、自动化 | 认知偏好、决策模式、价值体系 |
| 依赖 | 不依赖 Self-Distiller | **不依赖 LLM Wiki，可单独用** |

## 核心特色

### 1. 保留矛盾，不做简化标签

你说「投资只存款」又「裸辞创业」→ 标记为场景依赖矛盾，不做强行统一。
去年喜欢独处，今年喜欢社交 → 标注时间线，说明是成长而非矛盾。
自评「内向」但朋友评价「外向」→ 保留双方视角。

### 2. 每条认知有来源可查

> 「用户偏好表格对比信息（来源：2026-06-29 日记第 3 段 + 小红书笔记）→ 置信度：高」

你可以反问「为什么这么觉得？」AI 翻原始材料给你看。

### 3. 置信度体系

- **高** — 2 份+独立材料佐证
- **中** — 1 份材料 + 逻辑合理（Phase 2）
- **低** — 仅推测，回答时注明

### 4. Agent 无关

Hermes / Claude Code / Crush / 本地 Ollama — 任何能读写文件的 AI 都能操作。

## 快速开始

1. 把 `self-distiller/` 放到你的 AI 能访问的位置
2. 告诉 AI：「帮我装好 Self-Distiller，我要开始用」
3. AI 自动读完说明，初始化目录结构，告诉你准备好了
4. 开始导入材料！

---

## 使用方式

| 你说 | AI 做 |
|------|-------|
| 「把我这篇日记蒸馏进自我画像」 | 读材料 → 提取认知 → 写入 Wiki 条目 |
| 「我的决策风格是什么？」 | 翻 Wiki 回答你，注明来源 |
| 「帮我自检」 | 扫描所有条目 → 输出过期/低置信报告 |

| 操作 | Token 范围 | 说明 |

| 初始化 | ~500 | 读 SCHEMA + 创建目录和索引 |

| 查询 1 次 | 300~1,500 | 读 index + 目标条目 + 回答 |
