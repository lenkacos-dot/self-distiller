# Self-Distiller — 自我蒸馏知识库 Skill

> 基于 Karpathy LLM Wiki 模式的**特化实例**。
> 和 LLM Wiki 共用同一套架构（三层 + 增量工作流），只是领域不同：
> - LLM Wiki → 存你「拥有的知识」（工具、配置、项目）
> - Self-Distiller → 存你「是谁」（认知、决策、价值）
> 两者各自独立，互不依赖，可单独使用，也可搭配互补。

---

## 简介

让任何 AI Agent 持续编译、维护一个关于「你是谁」的结构化知识库。

- **输入**：日记、笔记、对话记录、任何自我相关材料
- **输出**：按维度组织的 Wiki 页面，含置信度、来源引用、矛盾标记
- **数据**：纯 Markdown + JSON，全存本地文件系统，不上传任何云端

---

## 工作原理

以 Karpathy LLM Wiki 模式为骨架，三层架构：

```
self-wiki/
├── SCHEMA.md              ← 维度模板、规则、Token 预算（本文件）
├── raw/                   ← 不可变层：原始材料，LLM 只读不写
│   └── material_index.md  ← 材料索引（ID → 摘要 → 来源）
├── wiki/                  ← Wiki 层：LLM 全权维护的结构化页面
│   ├── index.md           ← 目录
│   ├── 01-cognition.md    ← 认知偏好
│   ├── 02-decision.md     ← 决策模式
│   ├── ...  共 N 维
│   └── contradictions.md  ← 矛盾汇总（重点！）
├── .state/                ← 内部状态文件（用户不可见）
│   └── checkpoint.json    ← 断点 + Token 日志
└── reports/               ← 自检报告
    └── self-check/
```

## 维度体系

每个维度独立为一个 `wiki/NN-name.md`，格式统一。

### Phase 1（MVP）— 单维度

1. **认知偏好** — 学习方式、信息处理习惯、注意力模式

### Phase 2 — 扩展到 3 维

1. 认知偏好
2. **决策模式** — 高风险 vs 低风险、理性 vs 直觉、拖延 vs 果断
3. **价值体系** — 核心价值观、优先级、原则

### Phase 3 — 9 维全量

1. 认知偏好
2. 决策模式
3. 价值体系
4. **行为模式** — 日常习惯、社交风格、表达方式
5. **情绪模式** — 情绪触发、调节策略、韧性
6. **能力图谱** — 技能掌握的深度与广度
7. **关系模式** — 与他人的互动模式、信任节奏
8. **动机驱动** — 内在/外在驱动力、目标导向
9. **盲区 & 成长** — 已知不足、过往教训、无意识偏见

## 页面格式

每个 Wiki 条目统一结构：

```markdown
## 条目 ID: cognition-003

**置信度**: 高 | **创建**: 2026-06-01 | **更新**: 2026-06-15
**来源**: [raw/M003.md] | **版本**: 2（共更新 2 次）

### 内容
[核心描述]

### 证据
- [raw/M003.md] → 原文第 3 段：用户表示…
- [raw/M012.md] → 另一来源佐证同一倾向

### 变化历史
- v1 2026-06-01: 首次从日记 M001 提取
- v2 2026-06-15: 材料 M012 强化此观察，置信度从中→高

### 关联
- [[01-cognition#条目 cognition-001]]（同维度相关条目）
```

## Token 预算（硬约束）

以下限制在任何操作中必须遵守：

| 操作 | Token 上限 | 超限行为 |
|------|-----------|---------|
| 新材料解析 | 2000/批次 | 自动截断，记录断点 |
| Wiki 比对 | 3000/次 | 只比置信度"中"以上条目 |
| Wiki 写入 | 不设限 | 写入本身开销极低 |
| 自检扫描 | 5000/次 | 按置信度升序截断，低优先级跳过 |
| 查询回答 | 3000/次 | 优先高置信度条目 |

## 工作流

### A. 导入新材料

1. **接收材料** — 用户提供一份自我相关材料（日记、笔记、聊天记录等）
2. **写入 raw/** — 以材料 ID 保存到 raw/ 目录（不可变层）
3. **读取 index.md** — 了解已有条目
4. **解析提取** — 按当前维度体系提取关键信息
   - 已有维度的匹配条目 → 更新（加来源、调整置信度）
   - 新信息 → 创建新条目
5. **更新 contradiction.md** — 如果新信息与已有信息矛盾，记录矛盾（不做强行统一）
6. **更新 index.md** — 反映变更
7. **更新 checkpoint.json** — 记录本次 Token 消耗
8. **汇报** — 向用户说明本次导入的新增/变更条目

### B. 查询

1. 用户问关于自己的问题（"我的认知偏好是什么？"）
2. **读取 index.md** → 定位相关页面和条目
3. **读取对应条目** → 综合回答
4. **引用来源** → 说明基于哪些材料

### C. 自检（索引级，非全文回溯）

1. 扫描 wiki/ 下所有条目的置信度 + 最后更新时间
2. 标记：
   - 超过 30 天未更新的条目 → "可能过期"
   - 置信度为"低"的条目 → "需更多证据"
   - contradiction.md 中超过 30 天未处理的矛盾 → "待审视"
3. 输出自检报告（不自动修改条目）
4. 让用户决定是否对特定条目做全文回溯确认

## 矛盾处理（核心特色）

**不对矛盾做自动统一。** 保留矛盾并分类：

| 矛盾类型 | 示例 | 处理方式 |
|---------|------|---------|
| **时间演化** | "以前喜欢独处，现在喜欢社交" | 标注时间线，说明这是**变化**而非矛盾 |
| **场景依赖** | "工作场景理性，生活场景感性" | 标注场景标签，说明这是**切换**而非矛盾 |
| **认知盲区** | "自评内向，但好友评价外向" | 标记为**待审视**，建议用户自己判断 |

```markdown
## 矛盾: decision-002 vs decision-007

**类型**: 场景依赖 | **发现于**: 2026-06-15
**来源**: raw/M003.md（决策002）vs raw/M012.md（决策007）

**decision-002**: 风险偏好保守，"投资只存款不炒股"
**decision-007**: 冒险决策，"裸辞创业"

**分析**: 材料 M003 是银行理财场景，M012 是职业发展场景
**处理**: 标记为场景依赖矛盾，不统一。已建议用户补充更多职业决策材料。
```

## 与"蒸馏档案"的区别

这个 Self-Distiller ≠ 桌面上的 `🧪_Alan蒸馏档案_2026-06-29.md`：

| 维度 | 蒸馏档案（已有） | Self-Distiller（本 Skill） |
|------|----------------|--------------------------|
| **数据来源** | 一次性扫描历史会话 | 持续主动导入新材料 |
| **结构** | 长篇 Markdown 文档 | **条目化** Wiki（可增删改查） |
| **矛盾处理** | 无，统一叙述 | **保留矛盾**，分类标记 |
| **置信度** | 无 | 高/中/低 + 多来源强化 |
| **来源追踪** | 无引用 | 每条绑定材料 ID，可回溯 |
| **自检** | 无 | 索引级检查 + 过期提醒 |
| **Agent 无关** | 仅限 Hermes（session_search） | 任何 Agent 都能读写 |

## 与同类方案的对比

| 方案 | 存储形式 | 是否保留来源 | 是否保留矛盾 | Agent 无关 | 推荐场景 |
|------|---------|-------------|-------------|-----------|---------|
| **Hermes Memory** | 简短的 key-value | ❌ | ❌ | ❌ 仅 Hermes | 当前配置值 |
| **gbrain RAG** | 向量数据库 | ✅ | ❌ 检索混合 | ✅ | 通用知识检索 |
| **Clone/Persona Bot** | 模型微调或 Prompt | ❌ | ❌ | ❌ | 让 LLM"扮演"用户 |
| **LLM Wiki（我们的）** | Markdown Wiki | ✅ | ✅ | ✅ | 技术/项目知识 |
| **Self-Distiller（本方案）** | **结构化条目 Wiki** | **✅** | **✅ 矛盾分类** | **✅** | **自我认知** |

核心差异一句话：**别人的方案是让 AI 更了解「项目/文档」，Self-Distiller 是让 AI 更了解「你这个人」**——且保留你是一个复杂、矛盾、会变化的真实人类，而不是被简化成一个"用户画像标签集"。

## 隐私与安全

### 承诺
1. 所有数据仅存在本地文件系统，不上传任何云端服务
2. Wiki 条目不包含原始材料全文，仅通过材料 ID 引用
3. 用户可随时删除 self-wiki/ 目录彻底清除所有数据
4. 推荐将 self-wiki/ 加入 `.gitignore`，避免意外提交到公开仓库

### 风险告知
- 此 Skill 产出的数据高度个人化
- 请勿将 self-wiki/ 放入公有云同步目录（iCloud/Dropbox 需确认加密）
- 如与他人共享设备，建议为 self-wiki/ 启用文件加密
- 导入材料前，建议人工检查是否包含身份证号、银行卡号等强敏感信息

## 扩展建议（可选）

如果你正在使用 **LLM Wiki**（即 `~/llm-wiki/`），Self-Distiller 的 `self-wiki/` 与之互补：

```
~/llm-wiki/                ← 你「拥有的知识」（工具、项目、配置）
~/self-wiki/               ← 你「是谁」（认知、决策、价值、行为）
```

两个 Wiki 之间可以相互引用：`self-wiki` 说"用户偏好本地模型" → 可以引用 `llm-wiki/pages/devices.md` 里的华为 UOS 配置佐证。
