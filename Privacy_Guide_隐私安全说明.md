# Self-Distiller 隐私安全说明书（分享版模板）

> 本文件说明 Self-Distiller（数字生命蒸馏系统）的隐私保护机制及使用注意事项。
> **此版本为分享版模板，不含任何个人数据。导入你的材料后请根据实际情况修改。**

---

## 一、总体原则

Self-Distiller 的隐私安全分三层：

| 安全层级 | 保护目标 | 是否加密 |
|---------|---------|---------|
| raw/ 原始材料 | 简历、聊天记录、社交媒体原始数据 | ✅ 推荐 AES-256 加密 |
| wiki/ 知识维度 | 分析后的认知/语音/品味/记忆条目（不含完整原文） | ❌ 明文（供 AI 读取） |
| 分享版模板 | 指南 + 空白模板，不含任何个人数据 | ❌ 无个人数据 |

---

## 二、不同使用场景的安全风险

### 场景 1：使用云端 LLM（DeepSeek / OpenAI / Claude API）

- ⚠️ **wiki/ 内容会随每次提问发送至云端服务器**
- 这意味着：你的画像分析、人生记忆、语音风格等被 AI 服务商可见
- 风险等级：**中等** — 不含完整原文，但对方可能推断出你的身份
- 建议：
  - 不在 wiki 中写入完整姓名、电话、地址、身份证号
  - 重要对话摘要可仅保留关键风格/决策模式，删除具体人名地名

### 场景 2：使用本地 LLM（Ollama / LM Studio）

- ✅ 数据完全在本机运行，**不出网**
- 推荐长期使用 Self-Distiller 的隐私敏感用户采用

### 场景 3：分享版（本模板）

- ✅ 当前不含任何个人数据
- 你填写自己的数据后，请自行做好本地备份和隐私风险评估

## 三、raw/ 原始材料加密建议

### 推荐做法

1. 把你的原始数据文件放入 `raw/` 目录
2. 运行 `./encrypt-raw.sh lock` 加密
3. 密钥文件 `.enc_key` 会自动生成
4. 让 AI 解密 → 读取 → 分析 → 写入 wiki/ → 重新加密

### ⚠️ 重要：密钥备份

加密密钥文件位于项目根目录的 `.enc_key`。

**此文件丢失后，所有加密的原始材料无法恢复！**

建议将 `.enc_key` 备份到：
- 密码管理器（1Password / Bitwarden 等）
- USB 存储设备
- 云存储（注意做好访问控制）

## 三-B、加密能防什么，不能防什么

> AES-256-CBC 是军用级加密标准，但加密不是万能药。
> 理解它的边界，才能正确评估你的隐私风险。

### ✅ 加密能防的

| 威胁场景 | 加密能否防护 | 说明 |
|---------|------------|------|
| Git 误提交原始数据到 GitHub | ✅ 能防 | `.gitignore` + 后缀 `.enc` 双重阻挡 |
| 别人拿到你的硬盘读 raw/ | ✅ 能防 | .enc 文件是乱码，无密钥无法解密 |
| 云存储服务商扫描 | ✅ 能防 | 加密文件云端看到的是二进制乱码 |
| 电脑被盗后 raw 数据泄露 | ✅ 能防 | 只要密钥未随电脑一起拿到 |
| 共享屏幕/截图时 raw 内容外泄 | ✅ 能防 | 加密状态下不可读 |

### ❌ 加密不能防的

| 威胁场景 | 加密能否防护 | 说明 |
|---------|------------|------|
| AI Agent 读写 wiki/ 内容 | ❌ **不能防** | wiki/ 是明文（必须如此，AI 需要读取才能工作）。认知/语音/品味/记忆维度都会被发送到云端 LLM 服务器 |
| 云端 LLM 服务商查看数据 | ❌ **不能防** | 使用 DeepSeek/OpenAI API 时，wiki 内容会随每次提问被发送到其服务器 |
| 解密期间的内存泄露 | ❌ **不能防** | 运行 `unlock` 后，raw/ 文件以明文存在硬盘上，直到 `lock` 重新加密 |
| 操作系统级监控/键盘记录 | ❌ **不能防** | 加密不防护运行时的侧信道攻击 |
| AI Agent 推理出你的身份 | ❌ **不能防** | 即使 raw/ 加密了，wiki/ 中的记忆细节（家乡、学校、职业）仍可能推断出你是谁 |

### 🔑 核心结论

```
加密 = 保护"原始材料"（简历、微信原文）不被偷看
不加密 = 保护"分析结果"（wiki/ 中的画像）—— 因为分析结果必须被 AI 读取
```

**所以加密不是让你"完全安全"的，而是让你的原始数据不会因为一个误 git push 就全网公开。**

## 四、.gitignore 防误提交

仓库根目录的 `.gitignore` 会阻止以下文件被 git 追踪：

```
raw/         ← 所有原始材料（加密也要防误操作）
.enc_key     ← 加密密钥
```

建议添加你自己的其他敏感目录到 `.gitignore`。

---

## 五、首次使用清理清单

使用前，请确认：

- [ ] raw/ 目录下的原始材料已加密（文件后缀含 .enc）
- [ ] `.enc_key` 密钥已备份到安全位置
- [ ] `.gitignore` 存在且生效
- [ ] 使用云端 LLM 时，不在 wiki/ 中写完整姓名/电话/地址
- [ ] 如果你的版本要分享给别人，运行 `grep -r '你的姓名\|你的手机\|你的ID' .` 确认无个人信息残留

---

## 六、延伸建议

- **长期使用**：建议搭配本地模型（Ollama），数据不出网
- **临时使用云 API**：可以在 wiki/ 中把人名替换为代号（如 "User A"）
- **彻底卸载**：删除整个项目目录即可抹除所有数据
- **分享前**：运行 `find . -type f | xargs grep -l '你的真实姓名\|你的手机\|你的地址' 2>/dev/null` 确认无个人信息
