# Self-Distiller 隐私安全说明书

> 本文件说明 Self-Distiller（数字生命蒸馏系统）的隐私保护机制及使用注意事项。

---

## 一、总体原则

Self-Distiller 的隐私安全分三层：

| 安全层级 | 保护目标 | 是否加密 |
|---------|---------|---------|
| raw/ 原始材料 | 简历、微信聊天记录、社交媒体原始数据 | ✅ AES-256 加密 |
| wiki/ 知识维度 | 分析后的认知/语音/品味/记忆条目（不含完整原文） | ❌ 明文（供 AI 读取） |
| 分享版模板 | 填表指南 + 空白模板，不含任何个人数据 | ❌ 无个人数据 |

---

## 二、不同使用场景的安全风险

### 场景 1：使用云端 LLM（DeepSeek / OpenAI / Claude API）

- ⚠️ **wiki/ 内容会随每次提问发送至云端服务器**
- 这意味着：你的画像分析、人生记忆、语音风格等被 AI 服务商可见
- 风险等级：**中等** — 不含完整原文，但对方可推断出你是谁
- 建议：
  - 不在 wiki 中写入完整姓名、电话、地址、身份证号
  - 重要对话摘要可仅保留关键的风格/决策模式，删除具体人名地名

### 场景 2：使用本地 LLM（Ollama / LM Studio）

- ✅ 数据完全在本机运行，**不出网**
- 推荐长期使用 Self-Distiller 的隐私敏感用户采用

### 场景 3：分享版（self-distiller-skill/）

- ✅ 不含任何个人数据（已用 grep 验证：姓名/电话/ID/社交媒体 ID 均为 0 命中）
- 隐私说明已写入 SKILL.md 隐私安全表
- 用户填写后请自行做好本地备份和云 API 风险评估

## 三、raw/ 原始材料加密机制

### 加密范围

以下文件已被 AES-256-CBC 加密（`openssl enc -aes-256-cbc`）：

| 文件 | 内容 | 加密状态 |
|------|------|---------|
| M001-resume.md | 个人简历（真实姓名、电话、邮箱） | ✅ 已加密 |
| M002-wechat-high-signal.txt | 微信高信号对话片段（37K 条） | ✅ 已加密 |
| M003-xiaohongshu-tokensaver.md | Token Saver 小红书笔记 | ✅ 已加密 |
| material_index.md | 材料索引元数据（明文版本同时保留） | ✅ 已加密 + 明文 |

### 如何解锁与锁定

```bash
cd ~/Desktop/数字生命/self-distiller
# 解锁（解密到 raw/）
./encrypt-raw.sh unlock

# 编辑/读取原始材料...

# 重新锁定（加密 raw/）
./encrypt-raw.sh lock
```

### ⚠️ 重要：密钥备份

加密密钥文件位于：

```
~/Desktop/数字生命/self-distiller/.enc_key
```

**此文件丢失后，所有加密的原始材料无法恢复！**

建议将 `.enc_key` 备份到：
- iCloud Drive
- USB 存储设备
- 密码管理器

本系统已自动备份一份到：`~/Library/Mobile Documents/com~apple~CloudDocs/self-distiller-enc-key.backup`

## 三-B、加密能防什么，不能防什么

> AES-256-CBC 是军用级加密标准，但加密不是万能药。
> 理解它的边界，才能正确评估你的隐私风险。

### ✅ 加密能防的

| 威胁场景 | 加密能否防护 | 说明 |
|---------|------------|------|
| Git 误提交原始数据到 GitHub | ✅ 能防 | `.gitignore` + 后缀 `.enc` 双重阻挡 |
| 别人拿到你的硬盘读 raw/ | ✅ 能防 | .enc 文件是乱码，无密钥无法解密 |
| iCloud/iCloud Drive 扫描 | ✅ 能防 | 加密文件云端看到的是二进制乱码 |
| 电脑被盗后 raw 数据泄露 | ✅ 能防 | 只要密钥未随电脑一起拿到 |
| 共享屏幕/截图时 raw 内容外泄 | ✅ 能防 | 加密状态下不可读 |

### ❌ 加密不能防的

| 威胁场景 | 加密能否防护 | 说明 |
|---------|------------|------|
| AI Agent 读写 wiki/ 内容 | ❌ **不能防** | wiki/ 是明文（必须如此，AI 需要读取才能工作）。认知/语音/品味/记忆维度都会被发送到云端 LLM 服务器 |
| 云端 LLM 服务商查看数据 | ❌ **不能防** | 使用 DeepSeek/OpenAI API 时，wiki 内容会随每次提问被发送到其服务器 |
| 解密期间的内存泄露 | ❌ **不能防** | 运行 `unlock` 后，raw/ 文件以明文存在硬盘上，直到 `lock` 重新加密 |
| 操作系统级监控/键盘记录 | ❌ **不能防** | 加密不防护运行时的侧信道攻击 |
| AI Agent 推理出你的身份 | ❌ **不能防** | 即使 raw/ 加密了，wiki/ 中的记忆细节（家乡、学校、职业）仍可能推断出你是谁 |

### 🔑 核心结论

```
加密 = 保护"原始材料"（简历、微信原文）不被偷看
不加密 = 保护"分析结果"（wiki/ 中的画像）—— 因为分析结果必须被 AI 读取
```

**所以加密不是让你"完全安全"的，而是让你的原始数据不会因为一个误 git push 就全网公开。**

## 四、.gitignore 防误提交

仓库根目录的 `.gitignore` 会阻止以下文件被 git 追踪：

```
raw/         ← 所有原始材料（加密也要防误操作）
.enc_key     ← 加密密钥
wechat-export-work/  ← 微信导出临时文件
```

执行 `git add` 前请确认 `.gitignore` 存在且生效。

## 六、快速自检清单

首次使用前，请确认：

- [ ] raw/ 目录下的原始材料已加密（文件后缀含 .enc）
- [ ] .enc_key 密钥已备份到安全位置
- [ ] .gitignore 存在且生效
- [ ] 使用云端 LLM 时，不在 wiki/ 中写完整姓名/电话/地址
- [ ] 分享版不含个人数据（`grep -r '你的名字'` 确认）

## 七、延伸建议

- **长期使用**：建议搭配本地模型（Ollama），数据不出网
- **临时使用云 API**：可以在 wiki/ 中把人名替换为代号（如 "User A"）
- **彻底卸载**：删除整个 `~/Desktop/数字生命/` 目录即可抹除所有数据
- **分享前**：运行 `grep -r '你的姓名\|你的手机\|你的ID' self-distiller-skill/` 确认无个人信息残留
