# Academic Distillation — 学术蒸馏指南

> 两个目标：
> 1. **蒸馏你自己的学术写作风格** → 写入 self-wiki，让 AI 模仿你的学术腔
> 2. **蒸馏论文/文章的知识** → 写入 paper-wiki，写论文时直接引用

## Section A：学术写作风格蒸馏（人→wiki）

### 目标

从你的论文草稿、课程作业、导师批注、审稿意见中提取你的**学术写作指纹**，写入 self-wiki 的「说话风格」和「认知偏好」维度，让 AI 帮你写论文时保持腔调一致。

### 数据源

| 数据源 | 提取难度 | 价值 |
|--------|---------|------|
| 你自己写过的论文（终稿） | ★☆☆ | 核心样本——最纯正的你的学术腔 |
| 论文草稿 + 修改历史 | ★★☆ | 能看出你的初稿习惯和修改方向 |
| 导师批注/反馈（PDF/邮件） | ★★☆ | 导师反复纠正的点 = 你容易跑偏的地方 |
| 审稿意见 + 作者回复 | ★★★ | 高水平学术辩护的写作样本 |
| 课程作业/实验报告 | ★☆☆ | 入门级样本，低年级阶段 |
| 论文致谢/Acknowledgements | ★☆☆ | 学术人格的感性面（简练、真诚、致谢节奏） |

### 提取维度

这些信息提取后写入 self-wiki 的相应维度：

#### 写入「04-说话风格」维度

| 提取项 | 问什么 | 示例 |
|--------|-------|------|
| **论证结构偏好** | 三段式？倒金字塔？问题→方法→结果→讨论？ | "喜欢 IMRaD 结构，Discussion 部分习惯先总结再展开" |
| **intro 写法模式** | 宏观背景→文献缺口→本文贡献？还是直接问题→方法？ | "80% 论文以"领域背景→现存问题→本文方案"三段开篇" |
| **conclusion 风格** | 总结发现→局限性→未来工作？还是只总结不发散？ | "习惯总结→局限性→展望三段，但未来工作经常直接给方向而不铺垫" |
| **引用模式** | 密集引用（句末 [1-5]）？引导式（Smith et al. [2]）？评论式引用？ | "正文内引导式引用为主：'As X argued [3]...'，很少堆砌引用" |
| **数学/公式处理** | 公式嵌句中？单独编号行？符号命名风格？ | "公式单独编号右对齐，变量名偏好希腊字母+" |
| **制表习惯** | 数据放正文 vs 放附录？表格 vs 图？ | "结果数据首选图表，原始数据放附录" |
| **中英混用风格** | 学科术语用英文？全文英文写作？中文论文中的英文术语？ | "英文论文，但国内学术场合中英混杂，术语保持英文原词" |
| **hedging 语言** | "suggest/indicate/may/could" 的使用密度 | "倾向于弱结论：'These findings suggest that...' 而非 'This proves that...'" |
| **句式长度偏好** | 长复合句？短平快句？ | "平均句长 25-30 词，复合句占 60%，偏好并列从句而非嵌套从句" |
| **过渡词偏好** | "However/Moreover/Furthermore/Nevertheless" 的使用频率 | "however 和 moreover 占所有过渡词的 45%，nevertheless 几乎不用" |
| **致谢风格** | 致谢长度、语气、感谢顺序 | "致谢简练三段式：导师→合作者→基金，以个人口吻收官" |

#### 写入「01-认知偏好」维度

| 提取项 | 问什么 |
|--------|-------|
| **方法论偏好** | 定量/定性/混合方法？实验设计偏好类型？ |
| **论证风格** | 演绎（先原则后案例）vs 归纳（先数据后结论）？ |
| **文献组织方式** | 按主题分类？按时间线？按方法论阵营？ |
| **批判深度** | 温和指出不足 vs 尖锐批评？ |
| **跨学科倾向** | 引用常出没在哪个学科领域？ |

### 操作流程

```
1. 收集样本 → 自己的论文 PDF（最少 3 篇终稿）
2. 初稿草稿对比（如果有多个版本）
3. 导师批注/审稿意见（了解你的常犯错误模式）
4. 全部交给 AI → 按上述表格提取 → 写入 self-wiki
5. 验证：让 AI 写一段该学科的 intro → 你评分 → 迭代修正
```

## Section B：论文知识蒸馏（论文→paper-wiki）

### 目标

读论文→不靠记忆，靠**结构化笔记**。每篇论文蒸馏成一页，存入 `paper-wiki/papers/`，写 Literature Review 和 Related Work 时直接引用。

### 与 self-wiki 的区别

| | self-wiki | paper-wiki |
|---|---|---|
| 讲的是谁 | **你** | **论文/作者** |
| 用途 | AI 了解你 → 替你写 | 你了解论文 → 自己引用 |
| 知识类型 | 人的属性 | 论文的知识 |
| 引用场景 | "帮我按我的风格写" | "这 3 篇论文在 X 问题上的观点对比" |

### 每篇论文的蒸馏模板

每篇论文单独一个文件：`paper-wiki/papers/AuthorYear-ShortTitle.md`

```markdown
# AuthorYear-ShortTitle

> **完整引用**: Author, A., & Author, B. (Year). *Title of the paper*. Journal Name, Volume(Issue), Pages. DOI

## 元信息

- **标签**: `domain-X` `method-Y` `key-concept-Z`
- **阅读日期**: YYYY-MM-DD
- **相关论文**: [[AuthorYear-OtherPaper]]
- **重要程度**: ★★★☆☆

## 研究问题

一句话说清楚这篇论文在解决什么问题。

## 核心方法

- 方法/架构名称：
- 关键思路（3 行以内）：
- 数据集/实验设置：

## 主要发现

1. 发现 A（一句话 + 支持数据）
2. 发现 B
3. 发现 C

## 我对此文的评价

> 你觉得好在哪里、差在哪里、对你有用吗？

- 优点：
- 不足/可改进：
- 对我自己研究的启发：

## 可引用片段

> "此处是原文中可以直接引用的句子"（p.XX）

> "另一种表述角度"（p.YY）

## 引用关系

- **本文引用了**: [[AuthorYear-OtherPaper1]], [[AuthorYear-OtherPaper2]]
- **后来被引于**: （论文发表后再补充）
- **和我的研究关系**: 本文是关于 _____ 的核心参考文献 / 侧面参考 / 反对观点

## 关键图表

- 图 1（描述）
- 图 2（描述）
- 表 1（描述）

---

*蒸馏人*: Your Name | *蒸馏日期*: YYYY-MM-DD
```

### 操作流程

```
1. 拿到一篇新论文
2. 打开 paper-wiki/papers/XXX.md 模板
3. 让 AI 扫描 PDF → 按模板填充
4. 你快速扫一遍 + 补充"我对此文的评价"
5. paper-wiki/_index.md 更新索引
```

### 批量处理建议

- **单篇精读** → 使用完整模板
- **10-20 篇文献初步筛选** → 只填「研究问题 + 核心方法 + 我对此文的评价」三字段即可，读完再决定是否精读
- **Literature Review 阶段** → 按主题批量整理 `paper-wiki/topics/`，引用论文间的对比关系

## Section C：论文写作用例（组合应用）

### 用例 1：「帮我按我的风格写 Introduction」

组合：
1. 从 self-wiki → 查「04-说话风格」中的 intro 写法模式
2. 从 paper-wiki → 查相关论文的"研究问题"字段，了解背景
3. AI 用你的学术腔写引言，引用 paper-wiki 中的论文

### 用例 2：「帮我对比这 3 篇论文的方法论优劣」

组合：
1. 从 paper-wiki → 读 3 篇论文的「核心方法」+「评价」
2. AI 生成对比分析表格

### 用例 3：「帮我写 Related Work 段落」

组合：
1. paper-wiki → 按主题过滤论文
2. AI 按你的论证风格（主题分类/时间线/方法论阵营）组织段落
3. 引用格式自动填充

### 用例 4：「分析我论文的写作缺陷」

组合：
1. 上传你的论文草稿
2. AI 对比 self-wiki 中你的学术风格基线
3. 对比导师批注中纠正过的问题
4. 输出：哪些段落偏离了你自己的风格 + 哪些是导师之前指出的老问题

---

## 注意事项

- **paper-wiki 中的数据不加密**（所有论文蒸馏内容都是公开知识或你自己写的，不存在隐私问题）
- **paper-wiki 不应包含未授权复制的大段原文**——只保留"可引用片段"（短句引用，符合合理使用）
- self-wiki 部分仍然遵循 Self-Distiller 的加密策略：原始材料加密，蒸馏结果明文
- 建议在 `_index.md` 底栏加一条 paper-wiki 的快速入口链接
