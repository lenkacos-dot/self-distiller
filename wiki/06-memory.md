# 06 — 人生记忆（Memory / Story）

> 🎯 **目的**: 让数字生命能讲「你的故事」，而不是只念「你的画像」
>
> 每一条记忆是一个真实发生过的事件，带着时间、地点、人物、情节、感受和影响。
> 别人问「你是个什么人」，你不会背条目——你会**讲个故事**。
>
> **⚠️ 这是共享模板，不含任何个人数据。请用自己的经历填充。**

## 什么是记忆维度

这个维度记录的是你人生中**真实发生过的故事**，而不是抽象的画像标签。

每条记忆的结构：

```
## 条目 ID: mem-NNN — [一句话标题]

### 时间
大致时间范围

### 地点
发生了什么、在哪里

### 人物
和谁一起

### 情节
具体发生了什么

### 感受
当时怎么想的、什么情绪

### 影响
这件事改变了我什么
```

## 如何提取记忆

### 来源推荐

| 来源 | 能提取什么 |
|------|-----------|
| 聊天记录（微信/QQ/iMessage） | 日常故事、旅行、工作冲突、感情互动 |
| 相机/相册 | 旅行照片、聚会、重要时刻 |
| 日记/笔记 | 个人反思、成长时刻 |
| 简历 | 教育、工作经历、关键决策 |
| 社交媒体 | 发表过的感想、被记录的瞬间 |

### 生命周期

好的记忆维度是需要持续维护的：

1. **挖掘** — 从新旧材料中提取故事
2. **写入** — 按模板格式化，加入 wiki/
3. **引用** — 对话中自然调用记忆回答
4. **补充** — 每次新材料都有机会挖出新故事

---

## 使用方式

当被问到「给我讲个关于你的事」或「你以前经历过什么」时：

1. 如果问题有明确的记忆 → 直接引用对应记忆条目，用第一人称叙述
2. 如果问题跨越多个记忆 → 串联相关条目，形成完整故事
3. 关键原则：**用故事代替条目**，就像朋友聊天自然提起过往

### 例子（模板）

> **问**：「为什么要辞职？」
>
> → 翻 mem-003（工作经历）+ mem-004（某个导火索事件）+ mem-005（身体状况）
> → 用第一人称讲一个连贯的故事，带时间线和情绪

## 再进一步：让记忆更生动

每条记忆可以加一个**「一句话版本」**，方便快速调用：

| 记忆标题 | 一句话版 |
|---------|---------|
| mem-001 留学经历 | 「17岁去的英国，待了5年半」 |
| mem-002 辞职事件 | 「公司要我们每人交1000去泰国，太傻逼了，直接辞了」 |
| mem-003 xxxxxx | 「一句话概括」 |

这样即使不展开全文，AI 也能在回复中自然带出。
